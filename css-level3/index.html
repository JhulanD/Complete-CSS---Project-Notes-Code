CSS-Level3 - Units and Position

CSS Units

CSS units define how sizes and lengths are measured when styling HTML elements. They fall into two main categories:

1. Absolute Units

Absolute units have fixed sizes and do not scale based on the viewport or parent element. They are useful for precise control, especially in print or fixed layouts.

  • px - Pixels (most common for screens)
  • cm - Centimeters
  • mm - Millimeters
  • in - Inches
  • pt - Points (1pt = 1/72 inch)
  • pc - Picas (1pc = 12 points)

2. Relative Units

Relative units scale based on other factors, such as the font size of the parent or root element, or the size of the viewport. They are essential for responsive design.

  • em - Relative to the font size of the parent element
  • rem - Relative to the font size of the root (<html>) element
  • % - Percentage relative to the size of the parent element
  • vw - 1% of viewport width
  • vh - 1% of viewport height
  • vmin - 1% of the smaller viewport dimension
  • vmax - 1% of the larger viewport dimension

Example:

body {
  font-size: 16px;
}

h1 {
  font-size: 3rem; /* 3 times root font size */
}

.container {
  width: 80vw; /* 80% of viewport width */
  margin: 0 auto;
}

.button {
  width: 150px; /* fixed width */
  padding: 1em;
}

CSS Position

The position property specifies how an element is positioned in the document flow, and what reference point it uses for positioning.

Position values and their behavior:

  • static (default): Follows normal document layout flow.
  • relative: Positioned relative to its normal position.
  • absolute: Positioned relative to the closest positioned ancestor (non-static) or to the viewport if none is found. Removed from normal flow.
  • fixed: Positioned relative to the viewport and does not move when scrolling. Removed from normal flow.
  • sticky: Acts like relative until reaching a threshold, then behaves like fixed.

Position offsets

When an element is positioned (relative, absolute, fixed, sticky), you can use:

  • top
  • right
  • bottom
  • left

These define the offset from the respective edge of its containing block or viewport.

Example:

/* Relative positioning */
.relative-box {
  position: relative;
  top: 10px;
  left: 20px;
}

/* Absolute positioning */
.absolute-box {
  position: absolute;
  top: 50px;
  left: 100px;
}

/* Fixed positioning */
.fixed-header {
  position: fixed;
  top: 0;
  width: 100%;
  background-color: white;
}

/* Sticky positioning */
.sticky-nav {
  position: sticky;
  top: 0;
}

CSS background-size

The background-size CSS property specifies the size of the background image. It controls how the image scales inside the container.

Values

  • auto - The background image is displayed at its original size (default).
  • length - Set explicit width and height (e.g., 200px 100px). If only one length is given, the other defaults to auto.
  • percentage - Width and height as a percentage of the container size (e.g., 50% 30%).
  • cover - Scales the image to cover the entire container while preserving aspect ratio. Some cropping may occur.
  • contain - Scales the image to be fully visible within the container preserving aspect ratio. Might leave empty space.

Examples

auto

100px 50px

cover

contain

/* Original image size */
.element1 {
    background-image: url('image.jpg');
    background-repeat: no-repeat;
    background-size: auto;
}

/* Fixed size */
.element2 {
    background-image: url('image.jpg');
    background-repeat: no-repeat;
    background-size: 100px 50px;
}

/* Cover entire container */
.element3 {
    background-image: url('image.jpg');
    background-repeat: no-repeat;
    background-size: cover;
}

/* Image fully visible inside container */
.element4 {
    background-image: url('image.jpg');
    background-repeat: no-repeat;
    background-size: contain;
}

CSS Media Queries

CSS Media Queries allow you to apply different styles to a webpage depending on device characteristics, such as screen size, orientation, and resolution. This is essential for responsive web design.

Syntax

@media media-type and (media-feature) {
  /* CSS rules */
}

Common media types include all, screen, and print. Media features are conditions like min-width, max-width, orientation, and more.

Examples

Change background color based on viewport width:

body {
  background-color: tan;
}

@media screen and (max-width: 992px) {
  body {
    background-color: blue;
  }
}

@media screen and (max-width: 600px) {
  body {
    background-color: olive;
  }
}

Navigation menu layout changes on smaller screens:

/* Large screens */
.navbar ul {
  display: flex;
  list-style-type: none;
  padding: 0;
}

.navbar li {
  margin-right: 20px;
}

/* Small screens */
@media screen and (max-width: 600px) {
  .navbar ul {
    display: block;
    margin: 0;
  }
  
  .navbar li {
    margin: 10px 0;
  }
}

Notes: Media queries use breakpoints (e.g., 600px, 992px) to define when styles should change, enabling responsive layouts.
